2. Форма инициализации платежа (PHP)

Кодировка должна быть UTF-8

Для перехода на страницу оплаты необходимо сформировать следующую форму: 

<form method="POST" action="https://camelotpay.com/ru/merchant">
<input type="hidden" name="cp_shop_id" value="<?=$shop_id?>">
<input type="hidden" name="cp_order_id" value="<?=$order_id?>">
<input type="hidden" name="cp_order_sum" value="<?=$order_sum?>">
<input type="hidden" name="cp_currency_id" value="<?=$currency_id?>">
<input type="hidden" name="cp_order_description" value="<?=$order_description?>">
<input type="hidden" name="cp_signature" value="<?=$signature?>">

<input type="submit" name="process" value="send" />
</form>

Параметры формы

URL мерчанта
action
URL-адрес мерчанта
Идентификатор магазина
cp_shop_id
Идентификатор магазина зарегистрированного в нашей системе на который будет совершен платеж
Идентификатор платежа
cp_order_id
В этом поле продавец задает идентификатор покупки в соответствии со своей системой учета. Желательно использовать уникальный номер для каждого платежа. Идентификатор должен представлять собой любую строку длиной не более 8 символов, состоящую только из цифр.
Пример: 12345
Сумма платежа
cp_order_sum
Сумма платежа, которую продавец желает получить от покупателя. Сумма должна быть больше нуля, дробная часть отделяется точкой, количество знаков после точки - два знака.
Пример: 1.00
Валюта платежной системы
cp_currency_id
Валюта платежной системы. Пример: USD
Описание платежа
cp_order_description
Описание товара или услуги. Формируется продавцом. Строка добавляется в назначение платежа. Пример: \'My Test Description\'
Электронная подпись
cp_signature
Контрольная подпись, которая используется для проверки целостности полученной информации и однозначной идентификации отправителя. Пример: 9F86D081884C7D659A2FEAA0C55AD015A3BF4F1B2B0B822CD15D6C15B0F00A08§

3. ВЫБОР ДОСТУПНЫХ СПОСОБОВ ОПЛАТЫ (PHP)

После передачи платежной формы в CamelotPay покупатель может выбрать удобный для него способ оплаты. Продавец может ограничить список доступных способов оплаты или явно определить один из них. Для управления доступными способами оплаты используются параметры платежной формы cp_enabled_ps и cp_disabled_ps. Каждый из этих полей может присутствовать в платежной форме несколько раз и должен содержать один из способов оплаты.
Пример:
Этот заказ можно оплатить только PAY
<form method="POST" action="https://camelotpay.com/ru/merchant">
<input type="hidden" name="cp_shop_id" value="<?=$shop_id?>">
<input type="hidden" name="cp_order_id" value="<?=$order_id?>">
<input type="hidden" name="cp_order_sum" value="<?=$order_sum?>">
<input type="hidden" name="cp_currency_id" value="<?=$currency_id?>">
<input type="hidden" name="cp_order_description" value="<?=$order_description?>">
<input type="hidden" name="cp_signature" value="<?=$signature?>">
<input name="cp_enabled_ps[]" value="PAY">
<input type="submit" name="process" value="send" />
</form>
Пример:
Этот заказ можно оплатить всеми доступными платежными системами кроме PAY
<form method="POST" action="https://camelotpay.com/ru/merchant">
<input type="hidden" name="cp_shop_id" value="<?=$shop_id?>">
<input type="hidden" name="cp_order_id" value="<?=$order_id?>">
<input type="hidden" name="cp_order_sum" value="<?=$order_sum?>">
<input type="hidden" name="cp_currency_id" value="<?=$currency_id?>">
<input type="hidden" name="cp_order_description" value="<?=$order_description?>">
<input type="hidden" name="cp_signature" value="<?=$signature?>">
<input name="cp_disabled_ps[]" value="PAY">
<input type="submit" name="process" value="send" />
</form>
